# Chapter 2. 스위프트 시작하기

## 1. 기본 명명 규칙

### 🚫 이름을 짓을 때 사용할 수 없는 경우
- 스위프트에서 미리 정한 **예약어** 또는 **키워드**
- 해당 코드 범위 내에서 이미 사용 중인 이름과 **동일한 이름**
- 연산자로 사용될 수 있는 기호 (`+`, `-`, `/` 등)
- **숫자**로 시작하는 이름
- **공백**이 포함된 이름

### 📝 표기법 컨벤션
- **함수, 메서드, 변수, 상수**: 소문자로 시작 (카멜 표기법, `lowerCamelCase`)
- **클래스, 구조체, 익스텐션, 프로토콜, 열거형**: 대문자로 시작 (파스칼 표기법, `UpperCamelCase`)
- **대소문자 구별**: `var`과 `Var`은 서로 다른 것으로 취급됨

### 세미콜론 (`;`)
- 명령 구문 뒤에 세미콜론을 붙이는 것은 **선택 사항** (필수 아님)

> **NOTE**
> ```swift
> let var: String = "example"
> ```
> 위 코드는 `var`이 스위프트의 키워드이므로 상수 이름으로 사용할 수 없어 오류가 발생함.

---

## 2. 콘솔 로그

### `print()`
- 로그를 출력한 뒤, 줄바꿈 문자(`\n`)를 자동으로 삽입하여 줄을 바꿈

### `dump()`
- `print()`보다 더 **자세한 정보**를 출력
- 출력하려는 인스턴스의 내부 콘텐츠 구조까지 상세하게 보여줌

### 문자열 보간법 (String Interpolation)
- 변수나 상수 등의 값을 문자열 내에 포함하고 싶을 때 사용
- **형식**: `"문자열 \(변수또는상수)"`
- `CustomStringConvertible` 프로토콜을 준수하는 `description` 프로퍼티로 구현됨
    - *(추가 학습 필요: 구체적인 작동 원리에 대한 이해 필요)*

---

## 3. 주석 (Comments)

- 프로그램 소스코드에 **설명이나 정보**를 남기는 목적
- 마크업(Markup) 문법에 맞춰 메서드, 변수, 클래스 등에 주석을 작성하면, **퀵헬프(Quick Help)** 기능을 통해 다른 프로그래머가 문서처럼 확인 가능
    - **문서화**에 큰 도움이 됨

---

## 4. 변수와 상수

### 변수 (`var`)
- **키워드**: `var` 사용
- **선언**: `var 변수명: 데이터타입 = 값`
- **타입 추론**: 데이터 타입을 생략하면 컴파일러가 값에 따라 타입을 추론하여 지정
    - 단, 타입 추론 오류 시 원인을 찾는 데 시간이 오래 걸릴 수 있음
    - 처음부터 타입을 명시하는 것이 장기적으로 시간을 절약하고, 컴파일 속도도 빠를 수 있음

### 상수 (`let`)
- **키워드**: `let` 사용
- **선언**: `let 변수명: 데이터타입 = 값`
- 변수와 마찬가지로 데이터 타입 생략 가능
- **사용 이유**:
    1. **가독성**: 코드의 의도가 직관적으로 보임
    2. **안전성**: 원주율 등 공식적으로 불변하는 값이나, 입력값의 최대 크기(예: 100) 등 변경되면 안 되는 값에 사용하여 의미를 부여함

# Chapter 2. Swift（スウィフト）を始める

## 1. 基本的な命名規則

### 🚫 名前として使用できない場合
- Swiftであらかじめ定められた**予約語**または**キーワード**
- そのコード範囲内で既に使用されている既存の名前と**同一の名前**
- 演算子として使用される記号（`+`, `-`, `/` など）
- **数字**で始まる名前
- **空白**が含まれる名前

### 📝 表記法（コンベンション）
- **関数、メソッド、変数、定数**：小文字で始める（キャメルケース、`lowerCamelCase`）
- **クラス、構造体、エクステンション、プロトコル、列挙型**：大文字で始める（パスカルケース、`UpperCamelCase`）
- **大文字・小文字の区別**：`var`と`Var`は別物として扱われる

### セミコロン (`;`)
- 命令文の後ろにセミコロンを付けることは**選択事項**（必須ではない）

> **NOTE**
> ```swift
> let var: String = "example"
> ```
> 上記のコードは、`var`がSwiftのキーワードであるため、定数名として使用できずエラーになる。

## 2. コンソールログ

### `print()`
- ログを出力した後、改行文字（`\n`）を自動的に挿入して改行する

### `dump()`
- `print()`より**詳細な情報**を出力する
- 出力しようとするインスタンスの内部コンテンツ構造まで詳しく表示する

### 文字列補間 (String Interpolation)
- 変数や定数などの値を文字列内に含めて表現したい場合に使用
- **形式**：`"文字列 \(変数または定数)"`
- `CustomStringConvertible` プロトコルに準拠する `description` プロパティで実装される
    - *(要学習：具体的な動作原理について理解が必要)*

---

## 3. コメント (Comments)

- プログラムのソースコードに**説明や情報**を残す目的
- マークアップ（Markup）文法に合わせてメソッド、変数、クラスなどにコメントを作成すると、**クイックヘルプ（Quick Help）** 機能を通じて他のプログラマーがドキュメントのように確認可能
    - **ドキュメント化**において非常に役立つ

---

## 4. 変数と定数

### 変数 (`var`)
- **キーワード**：`var`を使用
- **宣言**：`var 変数名: データ型 = 値`
- **型推論**：データ型を省略すると、コンパイラが値に基づいて型を推論し指定する
    - ただし、型推論でエラーが発生した場合、原因特定に時間がかかることがある
    - 最初から型を明示する方が長期的には時間を節約でき、コンパイル速度も速くなる可能性がある

### 定数 (`let`)
- **キーワード**：`let`を使用
- **宣言**：`let 変数名: データ型 = 値`
- 変数と同様にデータ型の省略が可能
- **使用理由**：
    1. **可読性**：コードの意図が直感的に伝わる
    2. **安全性**：円周率など公式に不変な値や、入力値の最大サイズ（例：100）など、変更されてはならない値に使用して意味を付与する
