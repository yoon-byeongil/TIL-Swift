# Chapter 1. Swift

## Swift 개요
- **안전을 우선**: 프로그래밍 중 발생할 수 있는 실수를 방지하는 설계
- **컴파일 언어**: 기본적으로 컴파일 언어이지만, 플레이그라운드(Playground)를 통해 스크립트 언어처럼 코드 결과를 즉시 확인 가능
- **간결한 문법**:
  - 대부분의 문법에서 소괄호`()`, 세미콜론`;` 생략 가능
  - 단, **중괄호`{}`는 생략 불가**
  - 개발자마다 다양한 코딩 스타일 가능

---

## 스위프트의 언어적 특성

### 안전성 (Safety)
- guard 구문, 철저한 오류 처리, 강력한 타입 통제 등을 통해 버그 방지

### 신속성 (Fast)
- C언어(C, C++, Objective-C)를 기반으로 하여 빠른 컴파일 및 실행 성능 제공

### 더 나은 표현성 (Expressive)
- 개발자가 사용하기 편하고 가독성이 좋은 문법을 구현

### 객체지향 (Object-Oriented Programming)
- 컴퓨터 프로그램을 여러 개의 독립된 단위인 **'객체(Object)'**의 모임으로 파악
- 각각의 객체는 서로 메시지를 주고받고 데이터를 처리
- 프로그램을 유연하고 쉽게 변경할 수 있어 대규모 소프트웨어 개발에 적합
- 유지보수가 간편하고 직관적인 코드 분석 가능

### 함수형 (Functional Programming)
- 대규모 병렬처리에 유리
- 프로그램의 상태 변화 없이 데이터 처리를 수학적 함수 계산으로 취급하는 패러다임
- **특징**:
  - 객체지향/명령형은 '값'이나 '상태'의 변화를 중요시하지만, 함수형은 **함수 자체의 응용**을 중요시함
  - 함수에 전달된 인자 값만 결과에 영향을 주므로 상태 값을 갖지 않음 (Side-effect 없음)
  - 어떤 상황에서 실행하더라도 항상 같은 결과를 도출

# Chapter 1. Swift (日本語)

## Swiftの概要
- **安全性優先**: 安全を最優先に設計
- **コンパイル言語**: 基本的にはコンパイル言語だが、プレイグラウンド(Playground)を通じてスクリプト言語のように結果を即座に確認可能
- **簡潔な文法**:
  - ほとんどの文法で丸括弧`()`やセミコロン`;`の省略が可能
  - ただし、**波括弧`{}`は省略不可**
  - 開発者によって多様なコーディングスタイルが可能

---

## Swiftの言語的特性

### 安全性 (Safety)
- guard文、エラー処理、強力な型管理などを通じてバグを防止

### 迅速性 (Fast)
- C言語(C, C++, Objective-C)をベースにしており、高速なコンパイルと実行性能を提供

### 表現力 (Expressive)
- 使いやすく、可読性の高い文法を実装

### オブジェクト指向 (Object-Oriented Programming)
- プログラムを複数の独立した単位である**「オブジェクト(Object)」**の集まりとして把握
- 各オブジェクトは互いにメッセージをやり取りし、データを処理
- プログラムを柔軟かつ容易に変更できるため、大規模ソフトウェア開発に最適
- 保守が容易で、直感的なコード分析が可能

### 関数型 (Functional Programming)
- 大規模な並列処理に有利
- プログラムの状態変化なしに、データ処理を数学的な関数計算として扱うパラダイム
- **特徴**:
  - オブジェクト指向/命令型は「値」や「状態」の変化を重視するが、関数型は**関数自体の応用**を重視する
  - 関数に渡された引数のみが結果に影響を与えるため、状態を持たない (Side-effectなし)
  - どのような状況で実行しても常に同じ結果を導出
