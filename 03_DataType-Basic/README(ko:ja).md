# Chapter 3. 데이터 타입 기본

## 3.1 Int와 UInt
- **32비트 아키텍처**: Int32가 Int 타입, UInt32가 UInt 타입으로 지정됨
- **64비트 아키텍처**: Int64가 Int 타입, UInt64가 UInt 타입으로 지정됨

### 진수 표현
- **10진수**: 평소와 동일하게 작성
- **2진수**: 접두어 `0b`를 사용
- **8진수**: 접두어 `0o`를 사용
- **16진수**: 접두어 `0x`를 사용

## 3.2 Bool
- 참(true) 또는 거짓(false)
- `변수명.toggle()`로 참/거짓 값을 반전시킬 수 있음

## 3.3 Float과 Double
- **Double**: 64비트 부동소수 표현 (최소 15자리의 십진수 표현 가능)
- **Float**: 32비트 부동소수 표현 (6자리까지 표현 가능)
- 무엇을 쓸지 모르겠다면 **Double** 사용을 추천

> **NOTE**
> 10진수 표현이 아닌 다른 표현(지수 표현 등)이 보이는 이유는 10진수로 표현할 수 있는 한계를 넘어섰기 때문임. 반대로 코드를 작성할 때 해당 표현을 사용하여 값을 넣을 수도 있음.

> **TIP: 임의의 수(난수) 만들기**
> ```swift
> Int.random(in: -100...100)
> UInt.random(in: 1...30)
> Double.random(in: 1.5...4.3)
> Float.random(in: -0.5...1.5)
> ```

## 3.4 Character
- 단 하나의 문자를 의미
- 유니코드에서 지원하는 모든 언어 및 특수 기호 등을 사용할 수 있음 (한글 포함)

## 3.5 String
- Character와 마찬가지로 유니코드9를 사용할 수 있음
- 값의 앞뒤에 큰따옴표(`"`)를 사용
- `append()` 메서드를 사용하여 문자열을 이어붙일 수 있음
- `+` 연산자를 통해서도 문자열을 이어붙일 수 있음
- `name.count`: name에 해당하는 문자의 수를 셈
- 다양한 기능 활용은 `String.playground` 참고

### 3.5.1 특수문자 (제어 문자)
- `\n`: 줄바꿈 문자
- `\\`: 문자열 내에서 백슬래시(\)를 표현하고자 할 때 사용
- `\"`: 문자열 내에서 큰따옴표(")를 표현하고자 할 때 사용
- `\t`: 탭 문자 (키보드의 탭키 효과)
- `\0`: 문자열이 끝났음을 알리는 null 문자

## 3.6 Any, AnyObject와 nil

### Any
- 스위프트의 **모든 데이터 타입**을 사용할 수 있음
- 변수 또는 상수의 데이터 타입이 `Any`로 지정되어 있다면, 어떤 종류의 데이터 타입이든지 상관없이 할당 가능

### AnyObject
- **클래스의 인스턴스**만 할당할 수 있음 (9장에서 자세히 다룸)

> **NOTE**
> `Any`와 `AnyObject`는 될 수 있으면 **사용하지 않는 편이 좋음**. 데이터 타입은 최대한 명시하는 것이 안전함.

### nil
- '없음'을 나타내는 스위프트 키워드
- 변수 또는 상수에 값이 들어있지 않고 **비어있음**을 나타냄

# Chapter 3. データ型の基本

## 3.1 IntとUInt
- **32ビットアーキテクチャ**：Int32がInt型、UInt32がUInt型として指定される
- **64ビットアーキテクチャ**：Int64がInt型、UInt64がUInt型として指定される

### 進数表現
- **10進数**：通常通り記述
- **2進数**：接頭辞 `0b` を使用
- **8進数**：接頭辞 `0o` を使用
- **16進数**：接頭辞 `0x` を使用

## 3.2 Bool
- 真 (true) または 偽 (false)
- `変数名.toggle()` で真偽値を反転させることができる

## 3.3 FloatとDouble
- **Double**：64ビット浮動小数点表現（最低15桁の10進数を表現可能）
- **Float**：32ビット浮動小数点表現（6桁まで表現可能）
- どちらを使うか迷った場合は **Double** の使用を推奨

> **NOTE**
> 10進数表記ではない表現（指数表記など）が表示される理由は、10進数で表現できる限界を超えているため。逆にコードを書く際、その表現を使用して値を代入することも可能。

> **TIP：任意の数（乱数）を作る**
> ```swift
> Int.random(in: -100...100)
> UInt.random(in: 1...30)
> Double.random(in: 1.5...4.3)
> Float.random(in: -0.5...1.5)
> ```

## 3.4 Character
- たった一つの文字を意味する
- Unicodeでサポートされている全ての言語および特殊記号などを使用可能（ハングルも含む）

## 3.5 String
- Characterと同様にUnicode9を使用可能
- 値の前後にダブルクォート（`"`）を使用
- `append()` メソッドを使用して文字列を連結できる
- `+` 演算子を通じても文字列を連結できる
- `name.count`：nameに該当する文字数をカウントする
- 多様な機能活用は `String.playground` を参照

### 3.5.1 特殊文字（制御文字）
- `\n`：改行文字
- `\\`：文字列内でバックスラッシュ（\）を表現したい場合に使用
- `\"`：文字列内でダブルクォート（"）を表現したい場合に使用
- `\t`：タブ文字（キーボードのTabキーと同じ効果）
- `\0`：文字列の終了を知らせるnull文字

## 3.6 Any, AnyObjectとnil

### Any
- Swiftの **全てのデータ型** を使用できる
- 変数または定数のデータ型が `Any` に指定されていれば、どんな種類のデータ型でも関係なく代入可能

### AnyObject
- **クラスのインスタンス** だけを代入できる（9章で詳しく扱う）

> **NOTE**
> `Any` と `AnyObject` は、なるべく **使用しない方が良い**。データ型は可能な限り明示する方が安全である。

### nil
- 「無し」を表すSwiftのキーワード
- 変数または定数に値が入っておらず、**空であること** を表す
